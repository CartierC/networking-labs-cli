================================================================================
HOW TO INTERPRET RESULTS
================================================================================

  Ping 0% loss, <50ms    Healthy connectivity to a public host
  Ping 0% loss, >200ms   Reachable but high latency — investigate routing
  Ping 100% loss         Host unreachable OR ICMP blocked (check SG/NACL)
  Traceroute * * *       That hop filters ICMP — not necessarily a break
  Traceroute stops early Path is broken at the last responding hop

REAL-WORLD USE:
  - First check after an EC2 instance is launched or restarted
  - Validate connectivity after a VPC route table change
  - Identify where in the path a packet is being dropped
  - Confirm public internet reachability from a new subnet
